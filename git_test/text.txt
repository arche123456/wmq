创建一个本地仓库
git init
创建一个文件
text.txt
通过Vscode图形化页面暂存成功，但提交本地失败
也可以通过git add .来暂存本地
通过git status来查看是否有未暂存的文件
发现是需要在提交本地的时候必须要提交生成信息
可以通过git commit -m "生成信息"来提交本地

没有创建云仓库，在直接本体提交远程仓库的时候发现可以直接创建一个
尝试先在github中创建远程仓库，再在源代码管理中提交，发现提交失败
最后询问得知正确的顺序应该是先创建远程仓库，再将远程仓库clone下来
在clone的远程仓库基础上创建文件进行修改，不然本地仓库和远程仓库都是仓库并且互不相知

可以通过git pull wmq main直接下拉仓库，pull需要先有一个本地仓库，会拉取远程仓库并和本地仓库进行合并
但是git clone不需要有本地仓库，是直接拉取远程仓库不进行合并

也可以使用git pull/git push直接进行推拉

git reset --soft HEAD~1使用该指令可以撤销一次commit，但保留暂存区
git reset --mixed HEAD~1使用该指令可以撤销一次暂存和commit，但保留工作区的更改
git reset --hard HEAD~1使用该指令可以撤销一次暂存和commit，并删除工作区的更改
